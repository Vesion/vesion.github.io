从Ghost到Hexo

发表于 2015-05-11

关于个人博客搭建，加上这次，算是折腾三次了，从最一开始的Wordpress，到Ghost，再到这次的Hexo，也算是小小见证了个人博客解决方案的的变迁，个人博客也从程序员专属，发展到了如今能带给非专业人员极速轻量的搭建体验，技术也从原来的SSHA，LAMP发展到了Node.js + NoSQL，部署从原来的需要前后端通吃，渐渐转成傻瓜式命令静态渲染，作为互联网千万大军的一份子，每次看到新兴技术的兴奋度也渐渐减少。

关于个人博客搭建解决方案的讨论实在太多了，一开始自己还乐此不疲得参与进去，跟人争得面红耳赤，总是把自己的喜好吹得天花乱坠，把其他的贬得一文不值，现在想想也是脸红得不行：）。虽然我是个不折腾不舒服星人，但是享受的也仅仅是折腾的过程，折腾完了却总提不起兴致好好写一篇像样的博客，总觉得自己懂得太少，不好意思把一些东西分享出来，怕真正的高手笑话，这导致之前的博客搭起来后就搁置起灰。

所以作为这次（希望是最后一次）博客迁移的第一篇文章，就写写怎么搭Ghost和Hexo吧，而Wordpress因为时间太过遥远而且不符合当今极度追求轻量简便的政治正确就跳过了。

Ghost

####相关介绍

Ghost 是基于 Node.js 的开源博客平台，由前 WordPress UI 部门主管 John O’Nolan 和 WordPress 高级工程师 Hannah Wolfe 创立，所以Ghost和Wordpress还是有点血缘关系的，Ghost于2013年发布，由于上了Node.js这趟车，所以在开源社区还是引起不小的反响的，反正我当时看到她时是欣喜不已，终于可以放弃Wordpress这个沉重的家伙了。

相比Wordpress，Ghost的优点确实不少：

  • 基于Node.js，容易吸引开源社区的贡献
  • 架构简单，执行效率高，渲染快
  • 安装简单，部署简单
  • 使用简单，用Markdown在线写作发布
  • 可以使用自带的sqlite3

当然，目前来看，缺点也是挺明显的：

  • 技术太新，带来不稳定性
  • 对移动端支持不好
  • 对中文支持不好（不过有专门的中文社区）
  • Markdown 支持不好，一些语法失效
  • 云端存储支持不好

所以，个人感觉 Ghost 是介于 Wordpress 这种重量级产品，和Jekyll/Hexo这些静态博客产品之间的一种解决方案，对于一些想体验从服务器搭建到博客部署一套完整流程的人来说，Ghost 是一个极好的选择。

服务器

国内国外的云服务器供应商不少，这里就不过多介绍了，我选的是阿里云ECS，没啥特殊理由，就因为他当时有个大学生计划，打折：）至于服务器的配置，硬件流量啥的没啥好说的，系统个人推荐 CentOS 吧，如果不熟悉，可以看阿里人写的《linux就是这个范儿》，非常实用的一本 Linux 入门书，个人感觉比鸟哥的那本Linux更适合我：）。

对了，这里说下，国内的站点管理越来越严格了，还记得以前云服务器是不需要备案就能直接搭域名的，现在貌似不行了，需要备案了，不过都可以在线完成，相当简单，就连拍照都可以自己拍完上传。域名当然也需要备案，推荐到万网买。服务器域名到手后，可以好好把玩了，没事就ssh上去一阵折腾：）。

Ghost搭建

首先，这里强烈推荐Ghost中文社区，因为 Ghost 原版对中文编码支持得不是很好，所以推荐到中文社区下载中文集成版 Ghost，能省很多事。中文社区上也有比较详细的搭建教程，我这里的教程是精简版，不过可以确保能成功搭建在Mac OS X 10.11 和 CentOS 6.5 上，所以如果不想看长篇大论，可以往下看。

下面是在 CentOS 6.5 上搭建 Ghost 0.7.4 的过程：

  1. Node.js 安装

    Ghost 目前支持的Node.js版本是 v 0.10.x 或者 4.2.x，可以在官网这里找到相应的版本下载，当然也可以用Mac上的brew或CentOS上的yum安装相应的Node.js版本。

  2. Ghost 安装

    到中文社区这里找最新的Ghost中文集成版下载，可以省去很多后面兼容中文的麻烦。将压缩包解压到任意目录，例如 $HOME/ghost，然后进入目录 cd $HOME/ghost，用npm安装Ghost npm install --production，安装成功后执行 npm start 以debug模式启动Ghost，然后就可以在浏览器中访问 127.0.0.1:2368 查看Ghost演示博客了，也可以进入后台管理页面 127.0.0.1:2368/ghost。
