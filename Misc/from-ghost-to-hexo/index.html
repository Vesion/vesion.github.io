从Ghost到Hexo

发表于 2015-10-11   |   分类于 Misc   |  

前言

关于个人博客搭建，加上这次，算是折腾三次了，从最一开始的Wordpress，到Ghost，再到这次的Hexo，也算是小小见证了个人博客解决方案的的变迁历程，个人博客从程序员专属，发展到了如今能带给非专业人员极速轻量的搭建体验，技术从原来的SSHA，LAMP发展到了Node.js + NoSQL，部署从原来的需要前后端通吃，渐渐转成傻瓜式发布，而我对于新兴技术的兴奋度，却渐渐降低。

关于个人博客搭建解决方案的讨论实在太多了，技术好一点的，喜欢一切尽在掌控，对高度捆绑式博客框架嗤之以鼻，技术不好的，以及非专业人员，则更青睐没那么多技术细节，简单易上手的方案。一开始自己还乐此不疲得参与进去，跟人争得面红耳赤，总是把自己的喜好吹得天花乱坠，把其他的贬得一文不值，现在想想也是脸红得不行：）。虽然我是个不折腾不舒服星人，但是享受的也仅仅是折腾的过程，折腾完了却总提不起兴致好好写一篇像样的博客，这导致之前的博客搭起来后就搁置起灰。

所以作为这次（希望是最后一次）博客迁移的第一篇文章，就写写Ghost和Hexo吧，而Wordpress因为时间太过遥远而且不符合当今极度追求轻量简便的政治正确就跳过了。这篇文章不是技术博客，不会探讨Ghost和Hexo的实现，用到的后台管理技术，从模板到静态文件的渲染方式等，这些在他们的官网上都有非常详细开发文档，再加上自己也没有仔细扒过源码，所以就不献丑了。这篇文章最主要说说我为什么要从Ghost迁到Hexo，以及用精简的步骤记录下部署过程方便查阅。

当然原来的Ghost博客依然还在云服务器上跑着。

还有，这次我给我的博客起名为 Uncharted，它来自顽皮狗发布在PlayStation上的系列游戏，它的中文名你肯定听过——神秘海域。

Hexo

先来说Hexo，因为这是个很轻量级的博客框架，我在折腾完Ghost后，博客很长一段时间就闲置了，现在想想并不是忙到没时间写博客，而是没水平把一些想法组织成一篇通体流畅的精致文章，所以确实需要常写常练常看，这篇小白文章就当热手了，后面准备把 Compiler 和 DS & Algorithms 的一些技术文章写好。

说会Hexo，先说下我知道Jekyll/Hexo还是在Ghost之前，但是当时正值折腾病高峰期，就没有重视这类静态托管类型的博客框架。后来自己通过搭Ghost把前后端都走了一遍后，反而对自己的服务器产生一种不信任感，难以形容这种感觉，再加上Ghost的线上Markdown支持不是特别好，所以就萌生了托管到Github的想法。

Ghost

相关介绍

Ghost 是基于 Node.js 的开源博客平台，由前 WordPress UI 部门主管 John O’Nolan 和 WordPress 高级工程师 Hannah Wolfe 创立，所以Ghost和Wordpress还是有点血缘关系的，Ghost于2013年发布，由于上了Node.js这趟车，所以在开源社区还是引起不小的反响的，反正我当时看到她时是欣喜不已，终于可以放弃Wordpress这个沉重的家伙了。

相比Wordpress，Ghost的优点确实不少：

  • 基于最近很火的Node.js，容易吸引开源社区的贡献
  • 架构简单，执行效率高，渲染快
  • 安装简单，部署容易
  • 使用方便，用Markdown在线写作发布
  • 可以使用自带的sqlite3

当然，目前来看，缺点也是挺明显的：

  • 技术太新，带来不稳定性
  • 对移动端支持不好
  • 对中文编码支持不好（不过有中文社区提供支持）
  • Markdown 支持不好，一些语法失效
  • 云端存储支持不好

所以，个人感觉 Ghost 是介于 Wordpress 这种重量级产品，和Jekyll/Hexo这些静态博客产品之间的一种解决方案，对于一些想体验从服务器搭建到博客部署一套完整流程，但是又不想深入到太多细节的人来说，Ghost 是一个极好的选择。

服务器

国内国外的云服务器供应商不少，这里就不过多介绍了，我选的是阿里云ECS，没啥特殊理由，就因为他当时有个大学生计划，打折：）至于服务器的配置，硬件流量啥的没啥好说的，看个人所需了，系统推荐 CentOS ，如果不熟悉，可以看阿里自己人写的《Linux就是这个范儿》，非常实用的一本 Linux 入门书，个人感觉比鸟哥的那本Linux更适合我：）。

对了，这里说下，国内的站点管理越来越严格了，还记得以前云服务器是不需要备案就能直接搭域名的，现在貌似不行了，需要备案，不过都可以在线完成，相当简单，就连拍照都可以自己拍完上传。域名当然也需要备案，推荐到万网买。服务器域名到手后，可以好好把玩了，有事没事就ssh上去一阵折腾。
